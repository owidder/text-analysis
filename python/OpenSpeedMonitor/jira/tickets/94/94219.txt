Option "Hierachical name as content element's name."	"*Summary:*

New option: Display hierachical names for all Building Block Types in the diagram (header and all rows)



*As-Is:*

* No other displaying options are possible



*To-Be:*

* The user can choose between ""normal"" (=non-hierachical) and hierarchical names as criterion for displaying.



*Implementation Details:*

* Save in the actual state in the url

* Default is normal names

* If the user choose hierachical sorting, all Building Blocks are displayed with hierarchical name (grandparent (if exists) : parent (if exists) : parent (if exists) : ... : actual element) 



*UI:*

* Legend: Nothing is displayed

* Toolarea: Create a new main entry on the last position:

** Main entry: ""Display"" (German: ""Anzeige"")

** Sub-entries: Toogle-Button: ""Non-hierarchical names"" / ""Hierarchical names"" (German: ""Nicht-hierarchische Namen"" / ""Hierarchische Namen""

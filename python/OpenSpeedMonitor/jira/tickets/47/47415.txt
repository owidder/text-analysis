Error on partial import with composed iteraQl-Filter	"Reported by I-102:

*Reproduce:*
* Do a partial export with a composed iteraQl-statement (e.g. (@Strategic drivers=""operational"")|(@Strategic drivers=""mobile"") )
* Import this excel file
* General error message in the first import step

*Expected:*
No error

*Approach:*
* Some BuildingBlockTypes get masked because the comparison is false as the order of the subpredicates is not deterministic
* Define a deterministic ordering for the subpredicates in the FilterPredicate.java, i.e.

Ordering<FilterPredicate> nameOrdering = Ordering.natural().onResultOf(new Function<FilterPredicate, String>() {

        @Override
        public String apply(FilterPredicate input) {
          return input.getName();
        }

      });
      Collections.sort(this.subPredicates, nameOrdering);"		Bug	iteraplan PO	11/Feb/15 5:30 PM	17/Feb/15 10:25 AM					elastic	import	partial

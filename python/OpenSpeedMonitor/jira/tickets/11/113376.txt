Permission handling for Users (Server & Client)	"*Server:*

* Extend: {{api/administration/permissions}} with new key ""manageUsers"" (see No. 22 in https://iteraweb.iteratec.de/confluence/pages/viewpage.action?pageId=94798389 )

* The apis {{administration/users}} and {{administration/users/ID}} are only permitted for if ""manageUsers"" is true



*Client:*

* On initial start check for ""manageUsers""

* If this is false, hide tile ""Users""

* Prohibit direct url-access with the message ""GENERAL_PERMISSIONS_WARNING"""		New Feature	iteraplan Speedbird	27/Feb/17 4:14 PM	09/Mar/17 12:54 PM

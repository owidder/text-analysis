Create Single-Attribute-group-REST-API (Server)	"*Summary:*

Create Single-Attribute-group-REST-API on the Server for all attribute groups



*Details:*

* Use ""/api/atg/ID""

* Use the Single Element REST API ( https://www.iteraplan.de/doc/display/iteraplan55/Single+Element+API ) as blueprint for the http status codes and handling of REST commands

* JSON-structure is defined by {{/api/atg/}}



* {color:red}UPDATE: {color} Move of an attribute from one group to another and change position should be done by attributes-REST-API and not by atg-REST-API. Please extend attributes-REST-API



* error handling: Send a meaningful body and http status code for POST of a null-name or not-unique-name



* POST requires ""name"" - default behaviour for isTopLevelATG is false, default position is last position

*  {color:red}UPDATE: {color} ""lastModifyUser"",  ""lastModifyTime"", ""attributes"" cannot be POSTed or PUTed

* POST or PUT to ""position"" changes also positions of other attribute groups, so you always have a closed series of numbers

* DELETE: all attributes of the deleted attribute group are placed in the same order on the bottom of the default attribute group

* POST- and PUT-requests should return in the body the full modified or created object



JSON-structure:



{code:java}

{

    ""id"": NUM,

    ""name"": {

      ""en"": STRING,

      ""de"":  STRING

    },

    ""persistentName"":  STRING,

    ""description"": {

      ""en"":  STRING,

      ""de"": STRING,

    },

    ""isTopLevelATG"": BOOLEAN,

    ""position"": NUM,

    ""lastModifyUser"": STRING,

    ""lastModifyTime"": STRING,

    ""attributes"": [

      {

        ""id"": NUM,

        ""persistentName"": STRING

      },

     {},

     ....

    ]

  }

{code}

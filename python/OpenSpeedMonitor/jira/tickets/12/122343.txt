wrong cell format in excel causes exception in import	"Case:

 * export the datamodel in excel format

 * change a text value (for instance an enum value) to a numeric value; the cell value is then aligned to right

 * import this excel file



Result: an exception occurs



The import can be approved by reading the cell value inside a try-catch-block and assigning invalid value when an exception occurs.



Example in XlsMappingUtil.java:

{code:java}

private ElasticValue<ValueExpression> deserializeMultiValue(Cell cell, RValueTypeExpression<?> valueType) {

  // For multi value properties we always need the cell content as string, to split values. But we also can assume we won't have

  // date or numeric properties here, as those cannot be multivalue in iteraplan

  String stringValue;

  try {

    stringValue = cell.getStringCellValue();

  } catch (Exception ex) {

    // wrong cell format

    return ElasticValue.invalid();

  }

  if (StringUtils.isEmpty(stringValue)) {

    return ElasticValue.none();

  }

  ...

}

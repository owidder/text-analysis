Dies sollte durch die Konfiguration in der ""iteraplan-auth.properties"" durchgeführt werden können. Anbei die relevante Stelle:



""The roles filter that finds the roles where the user is a member of. The placeholder \{0} is replaced

# by the user object's DN. Alternatively, you can use \{1} to search with the user name.

# Most users can keep the default value, as 'member' is the most common attribute. Some directories

# use 'uniqueMember', though. 

#ldap.roles.membershipfilter=(member=\{0})

ldap.roles.membershipfilter=(member=\{0})""



Die genaue LDAP-Konfiguration/-Einrichtung ist jeweils unternehmensspezifisch und liegt außerhalb von iteraplan. Wir können Ihnen deshalb an dieser Stelle kein Beispiel nennen, oder bei der Konfiguration hier weiterhelfen. Hier sollte ein LDAP-Experte aus Ihrem Unternehmen die korrekte Abfrage eintragen können, wie die Gruppen / Rollen abgefragt werden.

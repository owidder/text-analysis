Don't write redundant update history entries for association types into standalone type history tables	"Example:

{noformat}

      ""changetype"": ""UPDATE"",

      ""changes"": [

        {

          ""featureName"": ""infrastructureElementAssociations"",

          ""featureType"": ""RELATIONSHIP"",

          ""isMultiple"": true,

          ""addedValue"": [

            ""Broker # 5.1 <-> Cluster 1""

          ]

        },

        {

          ""relation"": ""isr2Ie"",

          ""changesOfRelations"": [

            {

              ""idOfRelatedBB"": 350,

              ""changetype"": ""INSERT"",

              ""changes"": [ ... ]

            },

          ...

          ]

...

{noformat}



The whole first entry in the outer ""changes"" array is redundant, as the entry in the ""changesOfRelation"" array expresses the same change.



Please adjust {{HistoryEntryBuilder}} to not create update changes for new or removed attributable associations in a standalone type history entry."

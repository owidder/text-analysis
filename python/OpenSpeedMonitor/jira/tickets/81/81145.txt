Prozessvariablen in einem hierarchischen Kontext verwalten	"Der SecureBox-Prozess muss immer mehr Variablen speichern für die verschiedenen Aktionen (Spidern, Scannen, usw.). Um den Code übersichtlicher zu gestalten und die Aufgabenbereiche besser von einander zu trennen, könnten die Variablen stattdessen in einem hierarchischen Kontext (verschiedene Namensräume oder geschachtelte Objekte) verwaltet werden. Somit könnte jeder Kontext-Bereich bzw. -Namensraum die Konfiguration für eine bestimmte Aktion wie z. B. Spidern beinhalten. 

Dies beinhaltet auch, dass die Namen der Prozessvariablen an einer zentralen verwaltet werden sollten. Im Moment sind die Namen über die Process-Klassen und die Html-Seiten verteilt."

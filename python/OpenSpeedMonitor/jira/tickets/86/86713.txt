Es ist nicht möglich die Daten aus dem Lite Bundle von iteraplan 5.1 in ein Lite Bundle von iteraplan 5.2 zu übernehmen. Der Grund dafür ist, dass sich das Datenbank-Schema in der mitgelieferter HSQL-Datenbank für Version 5.2 geändert hat.
Mein Vorschlag wäre, damit Sie zukünftig nicht in dasselbe Problem hineinlaufen: Wenn Sie zukünftig mit einem corporate iteraplan anstatt eines Bundles arbeiten, können Sie zukünftig die Daten per Datenbankskript für mySQL, Oracle oder MS-SQL selbst migrieren.

"	"28/Jun/16 8:58 AM;msr;Hallo,

das können Sie so machen. Allerdings gibt es beim Import des Metamodells einige Einschränkungen, d.h. es werden nicht alle Details des Metamodells importiert. Siehe dazu auch: https://www.iteraplan.de/doc/display/iteraplan52/How+to+use+Import+and+Export und Unterseiten.

Einige Einschränkungen wären z.B. (ohne Anspruch auf Vollständigkeit):

""Please note ... that descriptions, sequencing and coloring of properties can not be imported and will be ignored.""
""Due to technical constraints there is currently no way to create attributes of the type ""Responsibility Attribute"" by adding the attribute to the excel file to be imported. When exporting landscape data or downloading the template, each responsibility attribute will be exported as text attribute. Importing data like this into a database where the according responsibility attribute already exists works correctly, but if the attribute does not exist already the attribute type and its values will be ignored: the attribute is not added to the building block type and the attribute values are not imported.""
""You cannot change whether an attribute is mandatory or not with the excel import.""
""The ordering of the children of a hierarchical element may be lost after importing.""

Wenn Sie mit diesen Import-Einschränkungen leben können, sollte es keine Schwierigkeiten geben.
Falls diese Import-Einschränkungen aber nicht passend für Sie sind, können Sie die fehlenden Informationen ""per Hand"" auf der Oberfläche nachtragen oder eine kostenpflichtige Migration beauftragen.

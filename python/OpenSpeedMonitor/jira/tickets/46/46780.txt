Linebreak for inner elements in Nesting Cluster diagram	"This ticket only affects the nesting cluster diagram:

In nesting cluster diagram, the name of the inner elements are shortened by ""..."", if the name is too long for the current box. Scope of this ticket is to implement a line break with maximum 2-3 lines for a building block name. 

*Solution:* If SVG 1.2 is used, there is a element named _textArea_, which can be used for a line break. If an SVG version is used which is older than 1.2, there are a few workarounds. The following example shows one solution to implement a multiline text. 

* The _tspan_ element, which is similar to HTML's _span_, can be used to format text. 

{noformat}
<?xml version=""1.0""?>
<!DOCTYPE svg PUBLIC ""-//W3C//DTD SVG 1.1//EN""
  ""http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd"">

<svg version=""1.1"" xmlns=""http://www.w3.org/2000/svg"">
  <text x=""10"" y=""40"" style=""font-size:30"">
    <tspan style=""font-weight:900"">S</tspan>
    <tspan style=""fill:silver"">V</tspan>
    <tspan style=""text-decoration:underline"">G</tspan>
    <tspan x=""10"" y=""75"">inkl. Zeilenumbruch</tspan>
  </text>
</svg>

Saved landscape diagrams cannot be loaded through menu form a BB overview page	"When the browser shows a building block overview page, e.g. Information Systems, it's impossible to load a saved landscape diagram through the Visualizations menu. It will always end up in a blank error page, as shown in the attached screenshot. Other diagram types do seem to work.

A short analysis showed that all form fields from the current page (i.e. the overview page) plus some added fields for loading the saved query are sent via a GET request. Some of the form fields cannot be matched to the _ManageLandscapeDiagramMemoryBean_ that is used for loading the query, and that leads to an exception.

A potential solution is that the JS function _preLoadSavedQuery()_ constructs a more targeted URI for loading the query, instead of using _flowActionRedirect()_. The edit/ delete quick action in the building block overview pages are probably a good example."

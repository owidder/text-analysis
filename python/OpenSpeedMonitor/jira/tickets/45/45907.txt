"06/Sep/13 1:52 PM;sry;The changes done in this ticket do actually address another issue and do not relate the problem mentioned in the description field! However, the described problem has been fixed for release 3.1 in issue ITERAPLAN-1612

In fact, the following changes ware made: During metamodel comparison of an imported file, metamodel changes are now only considered in an additive fashion, e.g. the import process will never attempt to delete any attributes. Hence it becomes legitimate to omit (enum) attribute sheets from the import file if they are irrelevant for the imported data. In particular, the special TypeOfStatus sheets for InformationSystems and TechnicalComponents can now also be omitted. This removes one common source of confusion."

WPT-Version im OSM persistieren	"Die Domain WebPageTestServer muss die aktuelle Version der WPT Instanz persistieren. 



Da sich die XML Struktur geändert hat, muss der OSM muss so angepasst werden, dass er mit der neuen und mit der alten Struktur umgehen kann.

Außerdem wird die Version von der Detail-Analyse benötigt, um entscheiden zu können wie die Daten geladen werden müssen. Die Version muss dann bei einem Request an die Detail-Analyse übergeben werden.

 

Seit der neuen Implementierung von WPT kann über das XML die WPT Version abgefragt werden.  Folglich müsste zunächst manuell eingetragen werden, welche die Version auf dem Server ist. Beim normalen Abfragen der Result XMLs können wir dann über das Feld zu der Version herausfinden, ob diese aktualisiert wurde. Solange das Feld nicht existiert, bleibt die Version gleich, ansonsten erhält die Domain die neue Version. Dieses update muss immer geschehen bevor die Detail-Analyse aufgefordert wird neue Daten zu laden."

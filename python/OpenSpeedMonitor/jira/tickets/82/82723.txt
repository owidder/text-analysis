Create Building-Block-Type-Selection for Level 2 and 3	"*Summary:*

Create Building-Block-Type-Selection for Level 2 and 3



*As-Is:*

* Building Block-Type-Selection is only possible for Level 1: ITERAPLAN-4524



*To-Be:*

* The user can select any related or self-related Building Block Type for Level 2 in the ToolArea and then another related or self-related Building Block Type for Level 3 in the ToolArea.



*Affected JS-Files:*

* mpdConfiguration

* mpdController (transferAction)

* mpdLegendController

* mpdLegendMain

* mpdLegendBuilder

* mpdToolAreaViewModelBuilder

* mpdContentController

* mpdContentMain

* mpdDataService



*Implementation Details:*

* This is the first ticket with with relations and self-relations:  Use the functionality of the metamodel to find out related Building Block Types_

** *Each self-relation and relation is available as ""relationshipEnds"" in the type.js*

* Use English terms also for the German localisation.



*UI:*

* Talk to [~msr] *before* starting this ticket

* Extend the Tool Area Main Entry called ""Content element type""

* Next level of the Tool Area (Drop-Down of Content element type) displays ""Level 1"", ""Level 2"" and ""Level 3""

* When you hover over ""Level 2"" all related Building Block Types and Self-relations are shown in an alphabetical order. The actual selection is highlighted in bold.

* At the beginning no ""Level 2""-entry is selected.

* When you hover over ""Level 3"", nothing is shown at the beginning. The entries appears after a ""Level 2"" is selected. The actual selection is highlighted in bold.

* Legend is created in ITERAPLAN-4522



*Acceptance Criteria:*

* The user opens a Masterplan Diagram.

* A default Masterplan Diagram is shown with Information System as Level 1

* The user clicks with primary click on any diagram element in the content.

* The Tool Area opens immediately

* The user hovers over ""Content element type"" in the Tool Area

* A Drop-Down opens immediately

* The user hovers over ""Level 2""

* Right of Level 2 an additional menu opens immediately which shows all related and self-related Building Block Types (see classical Masterplan Diagram)

* Nothing is highlighted

* The user selects ""Projects""

* The Tool Area is closed immediately, the diagram is redrawn

* Projects are now shown as Level 2 elements, the legend, the url and the tool area is updated



* * The user clicks with primary click on any diagram element in the content.

* The Tool Area opens immediately

* The user hovers over ""Content element type"" in the Tool Area

* A Drop-Down opens immediately

* The user hovers over ""Level 3""

* Right of Level 3 an additional menu opens immediately which shows all related and self-related Building Block Types of ""Projects"" (see classical Masterplan Diagram)

* Nothing is highlighted

* The user selects ""Business Processes""

* The Tool Area is closed immediately, the diagram is redrawn

* Business Processes are now shown as Level 3 elements, the legend, the url and the tool area is updated

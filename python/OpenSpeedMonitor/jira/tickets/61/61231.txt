Job-Queue-Darstellung: Eine Lane pro Test-Agent	"Für jeden Server die /getTesters.php aufrufen und die Anzahl der Test-Agenten pro Server ermitteln (die, deren LastWork kleiner x Minuten ist). 

Siehe z.B. http://dev.server02.wpt.iteratec.de/getTesters.php

Diesen REST-Call sollte man statt beim Aufruf der Job-Queue-Darstellung regelmäßig tun und die Anzahl der Agenten oder auch die einzelen Agenten als eigene Domain persistieren.



Pro Test-Agent innerhalb der Location-Darstellung eine hell farbig/grau hinterlegte Lane erzeugen.



Alle Job-Ausführungs-rects für die Darstellung ohne Überschneidungen auf diesen Lanes beliebig verteilen. Wenn das nicht mehr möglich ist sollten die Überlappungen rot dargestellt werden. Je mehr rects sich überlagern, desto dunkler rot.

Letzte Filtereinstellungen eines Benutzers vorhalten	"Wenn ein Benutzer eine Filterauswahl auf einem der Dashboards vorgenommen hat, so soll diese Einstellung gespeichert werden. Kehrt der Benutzer auf das jeweilige Dashboard zurück, so soll die letzte Filtereinstellung wiederhergestellt werden.

Hierzu notwenig ist eine generelle Benutzerverwaltung. Diese ist Möglich, durch die Verwendung des spring-security-plugins. Es müsste eine neue Rolle ""ROLE_User"" eingeführt werden. Eine Anmeldung ist das grundsätzlich erforderlich. An der Domainklasse User könnten die Einstellungen hinterlegt werden (diese ist allergings im ism plugin positioniert und daher in isocsi und isr nicht sichtbar, daher sollten im Vorwege die Plugins zu einem Projekt zusammengeführt werden ODER die Filtereinstellungen in anderen Domains platziert werden, die dann durch den Benutzer (Name oder Id) gefunden werden können).

Offene Frage: Wenn ein Benutzer mehere Tabs offen hat, kann das Speichern der Einstellungen verwirrend sein! Vielleicht eine Art Profil oder ""Meine Filter"" anbieten, so könnte der Benutzer aktiv das sichern von Einstellungen anfordern."

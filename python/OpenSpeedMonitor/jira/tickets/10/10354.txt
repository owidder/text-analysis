Änderung von Daten erhält älten Stand	"Werden Daten wie Page, WebPageTestServer, Browser oder Vergleichbares durch den Benutzer (per UI) geändert, so wird der ""echte"" Datensatz in der DB NICHT geändert. Stattdessen wird ein neuer Datensatz mit den jetzt aktuellen Daten angelegt. Dieser enthält eine Rückreferenz auf den Ausgangsdatensatz / die vorherige Version (Nachvollziehbarkeit von Änderungen).

Hierdurch wird erreicht, dass im Datenmodell ""gefahrlos"" mit direkten Referenzen gearbeitet werden kann und eine zunehmende Unübersichtlichkeit durch kopieren einzelner Felder verhindert wird."
